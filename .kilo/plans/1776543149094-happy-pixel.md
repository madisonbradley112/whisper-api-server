# Plan: Add Opus audio format support

## Analysis

Opus audio is stored in an OGG container. `python-magic` (libmagic) may detect it as `audio/ogg`, `audio/opus`, or `video/ogg`. The existing pipeline already handles OGG — FFmpeg on Ubuntu 24.04 supports opus decoding natively, and the audio processing pipeline (`convert_to_wav` → `normalize_audio` → `add_silence`) only works with intermediate WAV files, so no processing changes are needed.

## Changes (3 files)

### 1. `config.json` — add opus to allowed formats
- Add `.opus` to `allowed_extensions`
- Add `"audio/opus"` to `allowed_mime_types`

### 2. `app/infrastructure/validation.py` — normalize opus MIME
- Add `"audio/opus": "audio/ogg"` to `_MIME_EQUIVALENTS` dict (maps to already-allowed `audio/ogg`)

### 3. `app/audio/sources.py` — base64 detection for opus
- Add `"audio/opus": ".opus"` to `mime_to_ext` in `get_base64_file()`

## No changes needed
- **`Dockerfile`**: FFmpeg on Ubuntu 24.04 includes opus codec
- **`app/audio/processor.py`**: FFmpeg handles opus→WAV conversion already
- **`app/audio/utils.py`**: `get_audio_duration` uses ffprobe (opus-aware)
