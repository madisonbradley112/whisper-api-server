# Plan: Add 3GP Format Support

## Context

3GP files (e.g., `Voice 042.3gp`) are audio recordings from mobile phones. The test files in `~/Downloads/` contain AAC audio in a 3GPP container. FFmpeg already supports decoding these files (confirmed: `mov,mp4,m4a,3gp,3g2,mj2` demuxer, AAC codec). The `convert_to_wav()` pipeline in `app/audio/processor.py` uses FFmpeg, so no changes needed there.

## MIME Type

**Confirmed on orange server:** `python-magic` detects 3GP as `video/3gpp`. This needs to be normalized to `audio/mp4` (3GP is structurally an MP4 container with AAC audio, same mapping as `video/mp4` → `audio/mp4`).

## Changes (3 files)

### 1. `config.json` — Add `.3gp` extension
- Add `".3gp"` to the `allowed_extensions` array (line 30)
- **Important:** server has its own `config.json` — must also update it on orange via SSH after deploy

### 2. `app/infrastructure/validation.py` — Add MIME equivalent mapping
- Add `"video/3gpp": "audio/mp4"` to `_MIME_EQUIVALENTS` dict (line 20-27)
- No need to add a new MIME to `allowed_mime_types` since `audio/mp4` is already there

### 3. `app/audio/sources.py` — Add base64 format detection
- Add `"video/3gpp": ".3gp"` to the `mime_to_ext` dict (line 128-138)
